 <span data-ttu-id="40388-182">PNG 形式は、GIF 形式の利点の多くを保持しますが、GIF 以外の機能も提供します。</span><span class="sxs-lookup"><span data-stu-id="40388-182">The PNG format retains many of the advantages of the GIF format but also provides capabilities beyond those of GIF.</span></span> <span data-ttu-id="40388-183">GIF ファイルと同様に、PNG ファイルは情報を失わずに圧縮されます。</span><span class="sxs-lookup"><span data-stu-id="40388-183">Like GIF files, PNG files are compressed with no loss of information.</span></span> <span data-ttu-id="40388-184">PNG ファイルは、ピクセルあたり8、24、または48ビットの色を格納でき、グレースケールは1、2、4、8、または16ビット/ピクセルになります。</span><span class="sxs-lookup"><span data-stu-id="40388-184">PNG files can store colors with 8, 24, or 48 bits per pixel and grayscales with 1, 2, 4, 8, or 16 bits per pixel.</span></span> <span data-ttu-id="40388-185">これに対し、GIF ファイルで使用できるのは1、2、4、または8ビット/ピクセルだけです。</span><span class="sxs-lookup"><span data-stu-id="40388-185">In contrast, GIF files can use only 1, 2, 4, or 8 bits per pixel.</span></span> <span data-ttu-id="40388-186">PNG ファイルには、各ピクセルのアルファ値を格納することもできます。これは、そのピクセルの色を背景色とブレンドする度合いを指定します。</span><span class="sxs-lookup"><span data-stu-id="40388-186">A PNG file can also store an alpha value for each pixel, which specifies the degree to which the color of that pixel is blended with the background color.</span></span>  
  
 <span data-ttu-id="40388-187">PNG は、画像をプログレッシブに表示する機能 (つまり、ネットワーク接続を介して到着する画像をより正確かつ適切に表示するため) を強化します。</span><span class="sxs-lookup"><span data-stu-id="40388-187">PNG improves on GIF in its ability to progressively display an image (that is, to display better and better approximations of the image as it arrives over a network connection).</span></span> <span data-ttu-id="40388-188">PNG ファイルには、さまざまなディスプレイデバイスでイメージを正確にレンダリングできるように、ガンマ補正および色補正の情報を含めることができます。</span><span class="sxs-lookup"><span data-stu-id="40388-188">PNG files can contain gamma correction and color correction information so that the images can be accurately rendered on a variety of display devices.</span></span>  
  
### <a name="tag-image-file-format-tiff"></a><span data-ttu-id="40388-189">タグイメージファイル形式 (TIFF)</span><span class="sxs-lookup"><span data-stu-id="40388-189">Tag Image File Format (TIFF)</span></span>  
 <span data-ttu-id="40388-190">TIFF は、さまざまなプラットフォームやイメージ処理アプリケーションによってサポートされる、柔軟で拡張可能な形式です。</span><span class="sxs-lookup"><span data-stu-id="40388-190">TIFF is a flexible and extendable format that is supported by a wide variety of platforms and image-processing applications.</span></span> <span data-ttu-id="40388-191">TIFF ファイルは、ピクセルあたり任意の数のビットを持つイメージを格納でき、さまざまな圧縮アルゴリズムを使用できます。</span><span class="sxs-lookup"><span data-stu-id="40388-191">TIFF files can store images with an arbitrary number of bits per pixel and can employ a variety of compression algorithms.</span></span> <span data-ttu-id="40388-192">いくつかのイメージは、1つのマルチページ TIFF ファイルに格納できます。</span><span class="sxs-lookup"><span data-stu-id="40388-192">Several images can be stored in a single, multiple-page TIFF file.</span></span> <span data-ttu-id="40388-193">画像に関連する情報 (スキャナーの製造元、ホストコンピューター、圧縮の種類、向き、1ピクセルあたりのサンプルなど) は、ファイルに格納し、タグを使用して配置することができます。</span><span class="sxs-lookup"><span data-stu-id="40388-193">Information related to the image (scanner make, host computer, type of compression, orientation, samples per pixel, and so on) can be stored in the file and arranged through the use of tags.</span></span> <span data-ttu-id="40388-194">TIFF 形式は、新しいタグの承認と追加によって、必要に応じて拡張できます。</span><span class="sxs-lookup"><span data-stu-id="40388-194">The TIFF format can be extended as needed by the approval and addition of new tags.</span></span>
